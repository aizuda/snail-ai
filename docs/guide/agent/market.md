# 智能体市场

智能体市场（Agent Market）是 Snail AI 平台的智能体分发和共享机制。管理员可以将配置好的智能体发布到市场供组织内其他用户订阅使用，实现 AI 能力在企业内部的复用和推广。

<!-- screenshot: agent-market.png — 智能体市场页面，展示可订阅的公开智能体列表 -->

## 核心概念

| 概念 | 说明 |
|------|------|
| **我的智能体** | 当前用户创建或拥有的智能体列表 |
| **智能体市场** | 所有发布到市场的公开智能体列表，供全组织用户浏览和订阅 |
| **订阅** | 用户对市场中的智能体执行订阅操作后，该智能体将出现在其可用列表中 |
| **取消订阅** | 用户可以随时取消对某个智能体的订阅 |

---

## 我的智能体

通过 `/agent/my` 接口获取当前用户创建的全部智能体列表。

"我的智能体"视图展示用户自己创建的所有智能体，提供完整的管理权限：

- 查看和编辑配置
- 删除智能体
- 查看分析数据
- 设置精选推荐标记

---

## 浏览智能体市场

智能体市场（`/agent/market`）展示所有已发布的公开智能体。市场中的每个智能体卡片展示以下信息：

- **头像和名称**：快速识别智能体身份
- **描述**：了解智能体的功能定位
- **访问量**：反映该智能体的受欢迎程度
- **智能体 ID**：唯一标识，便于技术对接

用户可以浏览市场，发现和订阅适合自己业务需求的智能体，无需从零配置。

---

## 发布智能体到市场

要将智能体发布到市场，需要确保以下条件：

1. **基本信息完整**：名称和描述已填写
2. **配置就绪**：系统提示词已编写，必要的能力模块已绑定
3. **调试通过**：在对话预览区验证智能体表现符合预期

### 发布方式

在智能体详情页完成配置后，点击 **发布** 按钮保存配置，该智能体即进入可用状态。

如果希望重点推广某个智能体：

- 在配置页面开启 **精选推荐** 开关，使其在列表页的"企业精选"区域优先展示
- 发布到市场后，其他用户可在市场页面浏览并订阅

---

## 订阅智能体

当用户在市场中发现感兴趣的智能体时，可以执行订阅操作。

### 订阅流程

1. 在智能体市场中浏览可用智能体
2. 点击目标智能体查看详情
3. 执行 **订阅** 操作（`POST /agent/{id}/subscribe`）
4. 订阅成功后，该智能体出现在用户的可用列表中
5. 用户可直接与订阅的智能体进行对话

### 取消订阅

如果不再需要某个订阅的智能体，可以执行取消订阅操作（`DELETE /agent/{id}/subscribe`）。取消后，该智能体将从用户的可用列表中移除，但不影响其他已订阅的用户。

## "我的"与"市场"的区别

| 维度 | 我的智能体 | 智能体市场 |
|------|-----------|-----------|
| **数据来源** | 当前用户创建的智能体 | 全组织发布的公开智能体 |
| **管理权限** | 完整权限（编辑、删除、配置） | 仅可查看和订阅 |
| **API 端点** | `GET /agent/my` | `GET /agent/market` |
| **操作** | 创建、编辑、删除、发布 | 浏览、订阅、取消订阅 |
| **适用角色** | 智能体管理员 / 开发者 | 全组织用户 |

## 最佳实践

### 发布前检查清单

在将智能体发布到市场前，建议完成以下检查：

- [ ] 名称简洁明了，准确反映功能定位
- [ ] 描述清晰完整，用户阅读后能理解该智能体的价值
- [ ] 头像已设置，增强辨识度
- [ ] 系统提示词经过充分调优，测试结果稳定可靠
- [ ] 问候语引导到位，用户进入后知道如何开始
- [ ] 预设问题覆盖核心使用场景
- [ ] 知识库、MCP 工具等能力模块正常工作
- [ ] 在对话预览区完成多轮测试，包括正常场景和边界场景

### 命名规范建议

| 场景 | 推荐命名 | 不推荐命名 |
|------|----------|-----------|
| 财务分析 | 企业财报解读专家 | AI 助手 1 |
| 客服场景 | 飞书智能客服 | 客服机器人 |
| 数据分析 | 数据分析专家 | 数据 Bot |

建议命名包含**领域关键词** + **角色定位**，便于用户在市场中快速找到所需智能体。
