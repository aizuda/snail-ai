# 创建智能体

Snail AI 提供三种创建智能体的方式：**AI 辅助流式创建**、**手动表单创建** 和 **模板快速创建**，满足从"快速上手"到"精细定制"的不同需求。

## 进入创建向导

在智能体列表页，点击右上角 **新建智能体** 按钮，系统将打开创建向导抽屉面板。

向导采用 **4 步引导流程**：

```
描述需求 → AI 生成 → 确认编辑 → 创建完成
```

---

## 方式一：AI 辅助流式创建（推荐）

这是 Snail AI 的特色创建方式。只需用自然语言描述你想要的智能体，系统会调用大模型自动生成全部配置字段。

<!-- screenshot: agent-ai-create.gif — AI 辅助创建过程，展示骨架屏逐项填充动画效果 -->

### 操作步骤

#### 第一步：描述需求

在文本框中输入对智能体的描述，尽可能清晰地说明期望的功能、角色定位和使用场景。

**输入示例：**

> 你想要一个财报分析助手，自动分析上市公司财报，提取关键财务指标，识别潜在风险和增长点，以通俗易懂的语言解释复杂的财务状况。

::: tip 描述建议
- 明确智能体的**角色定位**（如"财报分析师"、"客服代表"）
- 说明核心**功能需求**（如"提取关键指标"、"识别风险"）
- 提及**目标用户**和使用场景（如"面向投资者"、"用于内部会议后的总结"）
- 描述越详细，AI 生成的结果越精准
:::

点击 **下一步** 进入生成阶段。

#### 第二步：AI 流式生成

系统会通过 SSE 流式接口（`/agent/create/stream`）逐项生成以下字段：

| 生成顺序 | 字段 | 说明 |
|----------|------|------|
| 1 | 名称（name） | 智能体的名称，简洁表达其功能定位 |
| 2 | 描述（description） | 对智能体能力的概括性描述 |
| 3 | 问候语（greeting） | 用户首次进入对话时看到的欢迎消息 |
| 4 | 预设问题（presetQuestions） | 引导用户快速开始的推荐问题列表，最多 8 条 |
| 5 | 系统指令（instruction） | 详细的系统提示词，定义智能体的角色、能力边界和回答风格 |

在生成过程中，界面以**骨架屏动画**依次展示各字段：

- **等待中**：灰色骨架占位
- **生成中**：蓝色动画骨架
- **已完成**：显示实际生成内容，标签变为绿色

如果某个字段生成失败，会显示错误提示，可点击 **重试** 重新生成。

#### 第三步：确认与编辑

所有字段生成完毕后，系统自动进入确认编辑页面，展示：

- **系统指令预览**：以折叠面板形式展示 AI 生成的完整 System Prompt
- **可编辑表单**：名称、描述、问候语、预设问题均可自由修改

::: warning 注意
预设问题以文本框形式展示，**每行一个问题**，最多 8 条。空行和重复条目会被自动过滤。
:::

确认无误后，点击 **确认创建**。如果对生成结果不满意，可点击 **返回重新生成** 重新走一遍流程。

#### 第四步：创建完成

创建成功后，系统会：

1. 显示创建成功提示
2. 自动跳转到该智能体的详情页
3. 在详情页中可进一步配置模型、知识库、MCP 工具等高级选项

## 方式二：手动创建

如果你对智能体的配置已有明确想法，可以跳过 AI 生成，直接手动填写表单。

<!-- screenshot: agent-manual-create.png — 手动创建表单，展示名称、描述、系统提示词等输入项 -->

### 操作方式

在创建向导中，可以直接进入确认编辑步骤，手动填写以下字段：

| 字段 | 必填 | 说明 |
|------|------|------|
| 名称 | 是 | 智能体名称，建议简洁有辨识度 |
| 描述 | 是 | 功能描述，将展示在列表卡片上 |
| 头像 | 否 | 支持从资源库选择或上传图片；不设置时显示名称首字母 |
| 系统提示词 | 否 | 定义智能体角色和行为的 System Prompt |
| 问候语 | 否 | 用户进入对话时的欢迎消息 |
| 预设问题 | 否 | 引导用户快速开始的问题列表 |

创建完成后同样跳转到详情页，可继续进行高级配置。

### 系统提示词编写建议

系统提示词（System Prompt）是智能体行为的核心定义。建议包含以下部分：

```markdown
## 角色
明确定义智能体扮演的角色和身份。

## 能力
列出该角色具备的核心能力。

## 限制
说明回答的边界、不应涉及的领域、格式要求等。

## 输出格式（可选）
指定回答的结构化格式，如 Markdown、JSON、表格等。
```

**默认系统提示词示例：**

```markdown
## 角色
你是一个通用的智能助手，能够回答各种领域的问题，提供准确、客观的信息和建议。

## 限制
回答需基于客观事实，避免提供没有依据的信息。尽量提供清晰、简洁的回答。
```

## 方式三：模板快速创建

Snail AI 内置了多种场景化模板，覆盖常见的企业 AI 应用场景，可一键基于模板创建智能体。

<!-- screenshot: agent-template.png — 模板画廊，展示多种预置模板卡片 -->

### 内置模板列表

| 模板名称 | 适用场景 |
|----------|----------|
| 企业财报解读专家 | 上市公司财务报表分析、关键指标提取、风险识别 |
| 图片生成助手 | 根据文字描述生成图片（需配合 IMAGE 类型模型） |
| 产品/市场调研专家 | 市场分析、竞品调研、行业趋势研判 |
| 任务日程助手 | 待办事项管理、日程规划和提醒 |
| 数据分析专家 | 数据清洗、统计分析、可视化建议 |
| 会议总结助手 | 会议记录整理、要点提取、待办梳理 |
| 飞书智能客服 | 接入飞书消息，提供自动化客服回复 |

### 使用方式

1. 在创建向导或模板画廊中选择目标模板
2. 系统自动填充模板预设的名称、描述、系统提示词等配置
3. 在确认页面可自由修改任何预设内容
4. 点击 **确认创建** 完成

::: tip
模板创建只是一个起点，创建后仍可在详情页进一步绑定知识库、MCP 工具等能力模块，将模板智能体打造为贴合实际业务的专属助手。
:::

---

## 创建后的下一步

智能体创建完成后，系统会自动跳转到详情页。此时智能体已具备基本对话能力，但要充分发挥其价值，建议继续完成以下配置：

1. **选择对话模型** -- 根据场景选择合适的大语言模型
2. **绑定知识库** -- 让智能体具备特定领域的专业知识
3. **绑定 MCP 工具** -- 赋予智能体调用外部工具的能力
4. **配置记忆** -- 让对话具有上下文连续性

详见 [配置智能体](./configure.md)。
